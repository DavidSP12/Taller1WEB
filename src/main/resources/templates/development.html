Desarrollo

Documentación técnica para desarrolladores, incluyendo guías de instalación, configuración del entorno, y referencias de API.

Guía de Instalación

Sigue estos pasos para configurar el entorno de desarrollo y ejecutar el proyecto localmente.

  • Paso 1: Clonar el repositorio con git clone https://github.com/usuario/proyecto.git
  • Paso 2: Instalar dependencias con mvn install
  • Paso 3: Configurar la base de datos según el archivo application.properties
  • Paso 4: Ejecutar la aplicación con mvn spring-boot:run

Stack Tecnológico

El proyecto utiliza las siguientes tecnologías y frameworks:

Java 11
Spring Boot
MySQL
Thymeleaf
Bootstrap
JavaScript
Git
Maven

Estructura del Proyecto

El proyecto sigue una arquitectura en capas con la siguiente estructura de directorios:

proyecto/
├── src/
│   ├── main/
│   │   ├── java/com/example/proyecto/
│   │   │   ├── controller/    # Controladores REST y MVC
│   │   │   ├── model/         # Entidades y DTOs
│   │   │   ├── repository/    # Interfaces de acceso a datos
│   │   │   ├── service/       # Lógica de negocio
│   │   │   ├── config/        # Configuraciones
│   │   │   └── util/          # Clases utilitarias
│   │   ├── resources/
│   │   │   ├── static/        # Recursos estáticos (CSS, JS)
│   │   │   ├── templates/     # Plantillas Thymeleaf
│   │   │   └── application.properties  # Configuración
│   └── test/                  # Pruebas unitarias e integración
└── pom.xml                    # Configuración de Maven

Endpoints API

La API REST expone los siguientes endpoints para interactuar con el sistema:

Usuarios

GET /api/users - Listar todos los usuarios

GET /api/users/{id} - Obtener usuario por ID

POST /api/users - Crear nuevo usuario

PUT /api/users/{id} - Actualizar usuario

DELETE /api/users/{id} - Eliminar usuario

Proyectos

GET /api/projects - Listar todos los proyectos

GET /api/projects/{id} - Obtener proyecto por ID

POST /api/projects - Crear nuevo proyecto

PUT /api/projects/{id} - Actualizar proyecto

DELETE /api/projects/{id} - Eliminar proyecto

Tareas

GET /api/tasks - Listar todas las tareas

GET /api/tasks/{id} - Obtener tarea por ID

POST /api/tasks - Crear nueva tarea

PUT /api/tasks/{id} - Actualizar tarea

DELETE /api/tasks/{id} - Eliminar tarea

Flujo de Trabajo

Seguimos un flujo de trabajo basado en Git Flow para el desarrollo y despliegue:

  • main: Rama principal que contiene código de producción estable.
  • develop: Rama de desarrollo donde se integran las nuevas características.
  • feature/*: Ramas para desarrollar nuevas funcionalidades.
  • release/*: Ramas para preparar versiones para producción.
  • hotfix/*: Ramas para corregir errores críticos en producción.

Para contribuir al proyecto, crea una rama feature/ desde develop, implementa tus cambios y envía un Pull Request.
