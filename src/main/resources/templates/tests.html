Pruebas

Documentación detallada sobre las estrategias de pruebas implementadas, incluyendo pruebas unitarias, de integración, y de aceptación.

Estrategia de Pruebas

Nuestra estrategia de pruebas se basa en la pirámide de pruebas, con un enfoque en pruebas automatizadas a todos los niveles:

Pruebas Unitarias
90%
Pruebas de Integración
75%
Pruebas E2E
60%
Pruebas Manuales
40%

Pruebas Frontend

Las pruebas del frontend se centran en verificar la correcta renderización de componentes, la interacción del usuario y la integración con el backend.

Pruebas Unitarias

Utilizamos Jest para probar componentes individuales y funciones utilitarias, asegurando que cada unidad funcione correctamente de forma aislada.

Pruebas de Componentes

Verificamos que los componentes se rendericen correctamente con diferentes props y estados, y que respondan adecuadamente a las interacciones del usuario.

Pruebas de Navegación

Comprobamos que las rutas y la navegación entre páginas funcionen correctamente, manteniendo el estado de la aplicación según lo esperado.

Pruebas Backend

Las pruebas del backend verifican la lógica de negocio, el acceso a datos y la exposición correcta de APIs.

Pruebas Unitarias

Utilizamos JUnit y Mockito para probar servicios y componentes individuales, aislando las dependencias mediante mocks.

Pruebas de Repositorio

Verificamos que las operaciones de acceso a datos funcionen correctamente, utilizando bases de datos en memoria para pruebas.

Pruebas de API

Comprobamos que los endpoints REST respondan correctamente a diferentes solicitudes y devuelvan los códigos de estado y datos esperados.

Pruebas de Integración

Las pruebas de integración verifican que los diferentes componentes del sistema funcionen correctamente juntos.

  • API a Base de Datos: Verificamos que los controladores interactúen correctamente con los servicios y repositorios.
  • Frontend a Backend: Comprobamos que las llamadas desde el frontend a las APIs del backend funcionen según lo esperado.
  • Servicios Externos: Verificamos la integración con servicios externos mediante mocks o entornos de prueba.
  • Flujos Completos: Probamos flujos de usuario completos que atraviesan múltiples componentes del sistema.
