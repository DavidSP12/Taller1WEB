Despliegue

Documentación detallada sobre los procesos de despliegue, configuración de entornos y monitorización del sistema en producción.

Entornos de Despliegue

El sistema se despliega en diferentes entornos según la fase del ciclo de desarrollo:

Desarrollo

Entorno local para desarrolladores donde se implementan y prueban nuevas funcionalidades. Utiliza una base de datos H2 en memoria para facilitar las pruebas.

Pruebas

Entorno de integración continua donde se ejecutan pruebas automatizadas. Utiliza una base de datos MySQL de prueba con datos de ejemplo.

Preproducción

Entorno idéntico a producción pero aislado, donde se realizan pruebas finales antes del despliegue. Utiliza una réplica de la base de datos de producción.

Producción

Entorno final accesible por los usuarios. Utiliza una base de datos MySQL optimizada y con alta disponibilidad.

Proceso de CI/CD

Utilizamos un pipeline de integración y despliegue continuo para automatizar el proceso de entrega:

  • Integración Continua: Cada commit a la rama principal desencadena la ejecución de pruebas automatizadas.
  • Análisis de Código: SonarQube analiza la calidad del código y detecta posibles vulnerabilidades.
  • Construcción de Artefactos: Maven compila el código y genera los artefactos necesarios para el despliegue.
  • Despliegue Automático: Jenkins despliega automáticamente en el entorno correspondiente según la rama.
  • Pruebas Post-Despliegue: Se ejecutan pruebas de humo para verificar que el despliegue fue exitoso.

Configuración

La configuración de la aplicación se gestiona de forma externa para facilitar los cambios entre entornos:

# application-prod.properties

# Configuración de la base de datos
spring.datasource.url=jdbc:mysql://${DB_HOST}:${DB_PORT}/${DB_NAME}
spring.datasource.username=${DB_USERNAME}
spring.datasource.password=${DB_PASSWORD}

# Configuración de JPA
spring.jpa.hibernate.ddl-auto=none
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL8Dialect

# Configuración de seguridad
security.jwt.secret=${JWT_SECRET}
security.jwt.expiration=86400000

# Configuración de servidor
server.port=8080
server.compression.enabled=true

# Configuración de logging
logging.level.root=WARN
logging.level.com.example=INFO
logging.file.name=/var/log/application.log
